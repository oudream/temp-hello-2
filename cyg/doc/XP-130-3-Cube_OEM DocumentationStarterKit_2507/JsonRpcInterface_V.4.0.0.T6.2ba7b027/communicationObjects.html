JSON-RPC Protocol Specification

Introduction

This document specifies the communication object entities, which are accessible via jsonRPC. The lower-level protocol of how exactly to read and write the objects (using jsonRPC and websocket) is specified in the document "Communication Protocol Specification".

Data Objects

The table below contains the accessible data objects. All data objects are json-arrays.

  • Resource: jsonRPC resource name
  • Maximum Length: maximum number of array elements to expect or supported, when reading/writing
  • Type: json data type
  • Const: whether the values are constant during runtime or not
  • Persistence: whether a modification of the value will persist through a reboot or not. user: persist on user write access, userConfig: persist from user config file
  • Access: r: read-only, w: write-only, rw: read and write
  • Description: detailed description
